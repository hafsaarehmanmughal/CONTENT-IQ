==============================================================
  STEP 1 — INSTALL NODE.JS (if not already installed)
==============================================================

1. Go to: https://nodejs.org
2. Download and install the LTS version (e.g. v22)
3. Verify installation by opening terminal and typing:
      node --version
      npm --version
   You should see version numbers printed.


==============================================================
  STEP 2 — OPEN TERMINAL IN YOUR BACKEND FOLDER
==============================================================

Windows:
  - Open File Explorer → go into your backend/ folder
  - Click the address bar, type: cmd → press Enter
  - OR right-click inside folder → "Open in Terminal"

Mac/Linux:
  - Open Terminal
  - Type: cd path/to/your/ContentIQ/backend
  - Press Enter


==============================================================
  STEP 3 — INSTALL DEPENDENCIES
==============================================================

In the terminal (inside backend/ folder), run:

   npm install

Wait for it to finish. You will see a node_modules/ folder appear.
This installs: express, prisma, bcryptjs, jsonwebtoken, cors, dotenv.
ALL FREE. No credit card. No paid services.


==============================================================
  STEP 4 — SET UP THE DATABASE
==============================================================

Run these two commands one by one:

   npx prisma generate

   npx prisma db push

This creates a file called dev.db inside your prisma/ folder.
That file IS your database. SQLite — free, local, no server needed.

You should see: "Your database is now in sync with your Prisma schema"


==============================================================
  STEP 5 — CONFIGURE YOUR .env FILE
==============================================================

Open backend/.env in any text editor (Notepad, VS Code, etc.)

It looks like this:

   DATABASE_URL="file:./prisma/dev.db"
   JWT_SECRET=contentiq_jwt_secret_change_this_2026
   GEMINI_API_KEY=your_gemini_api_key_here
   PORT=5000

The only thing you NEED to change is GEMINI_API_KEY for the
Content Humanizer service. Here's how to get it FREE:

   a. Go to: https://aistudio.google.com
   b. Sign in with your Google account
   c. Click "Get API Key" → "Create API Key"
   d. Copy the key (starts with "AIza...")
   e. Replace "your_gemini_api_key_here" with your key

Example:
   GEMINI_API_KEY=AIzaSyAbc123def456ghi789...

NOTE: If you skip this step, the Content Humanizer will still
work using a rule-based fallback — it just won't be as smart.
ALL OTHER 5 SERVICES WORK WITHOUT ANY API KEY.


==============================================================
  STEP 6 — START THE SERVER
==============================================================

In your terminal (inside backend/ folder), run:

   npm start

You should see:

   🚀 ContentIQ server running at http://localhost:5000
   📁 Serving frontend from: .../frontend
   🔗 API available at: http://localhost:5000/api


==============================================================
  STEP 7 — OPEN THE APP IN YOUR BROWSER
==============================================================

Open any browser and go to:

   http://localhost:5000

You will see the ContentIQ landing page.


==============================================================
  STEP 8 — TEST THE FULL FLOW
==============================================================

1. Click "Get Started" or go to /register
2. Create an account with your name, email, password
3. You will be redirected to Login — sign in
4. You land on your Dashboard
5. Click "Start New Analysis"
6. Paste any text (at least 100 words for best results)
7. Click "Continue to Services"
8. Select one or more services
9. Click "Analyze Now"
10. Wait for analysis (5-15 seconds)
11. View your real results on the Result page
12. Check Report History for all your past analyses


==============================================================
  HOW EACH SERVICE WORKS (WHAT MAKES IT ACCURATE)
==============================================================

SERVICE 1: SEO Optimization
  Method:  Pure math — word count, keyword density (TF), 
           sentence length, heading structure, paragraph analysis
  Accuracy: ~85% — comparable to Yoast SEO free
  API needed: NONE

SERVICE 2: Grammar & Readability  
  Method:  Flesch-Kincaid Reading Ease formula (industry standard),
           passive voice detection, sentence complexity analysis
  Accuracy: ~80% — comparable to Hemingway App
  API needed: NONE

SERVICE 3: Keyword Analysis
  Method:  TF-IDF algorithm (standard NLP), keyword distribution
           analysis, vocabulary richness scoring, LSI detection
  Accuracy: ~80%
  API needed: NONE

SERVICE 4: AI Content Detection
  Method:  Statistical analysis — sentence length variance,
           burstiness, contraction usage, AI transition words,
           vocabulary uniformity (research-based signals)
  Accuracy: ~70-75% — comparable to GPTZero free tier
  API needed: NONE

SERVICE 5: Similarity Check (Plagiarism)
  Method:  N-gram fingerprinting, common phrase detection,
           internal repetition analysis, vocabulary richness
  Accuracy: ~30-40% — detects structural copying and clichés
           NOT full internet plagiarism (that requires Turnitin)
  API needed: NONE

SERVICE 6: Content Humanizer
  Method:  Google Gemini 1.5 Flash AI (with your free API key)
           Falls back to rule-based replacement without key
  Accuracy: ~75-80% with Gemini key, ~50% with fallback
  API needed: FREE Gemini key from aistudio.google.com


==============================================================
  TROUBLESHOOTING
==============================================================

Problem: "Cannot find module" error when starting server
Solution: Run "npm install" again inside the backend/ folder

Problem: "prisma is not recognized" 
Solution: Run "npx prisma generate" then "npx prisma db push"

Problem: Server starts but browser shows "Cannot GET /"
Solution: Make sure your frontend/ folder is at the same level
          as backend/ (i.e., ContentIQ/frontend/ exists)

Problem: Login says "Could not connect to server"
Solution: Make sure "npm start" is running in a terminal.
          The terminal must stay open while you use the app.

Problem: Analysis takes very long or times out
Solution: Normal for first run. Subsequent analyses are faster.
          If using Gemini, check your API key is correct in .env

Problem: "Port 5000 already in use"
Solution: Change PORT=5001 in .env and restart server
